# Security Policy

## Rules
- Never commit secrets
- Never commit API keys, tokens, passwords or private keys
- Never commit broker data, statements, raw exports or personal data
- Never post sensitive screenshots in public issues

## If a Secret Is Suspected
- Stop commit and push immediately
- Rotate the secret
- Remove it from staged changes and local history before sharing

## Reporting
- Report security issues privately to the repository owner
- Do not open public issues containing keys, tokens, screenshots or account data

## Repository Scope
- Research dashboard only
- No live execution
- No broker connectivity
- No order execution
